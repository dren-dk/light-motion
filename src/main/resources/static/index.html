Dropwizard+AngularJS

Welcome to my dropwizard + angular + webjars seed project, this project is a mix of:

  • The Dropwizard tutorial
  • The AngularJS tutorial
  • Example of how to inject singletons and per-request objects using hk2
  • Example of how to use webjars rather than rely on The Entire Internet for your client side library needs as the Javascript folks like to do, which is strangely hard to find example code for.

The intention is that this project is simple enough to allow you to understand why everything is here and to allow you to simply copy the project and use it as a starting point for a new dropwizard project, it's not aiming to show off all dropwizard or angular features or even serve as an example of "The Right Way" of doing things.

Contents of the web scope

  • /swagger/: Swagger
  • /admin/: Dropwizard admin, note that a real, public server should have admin living on a separate, firewalled port
  • /static/: Static files served from the classpath, see resources/static/ in the source tree.
  • /webjars/: Webjars
  • /: This front page is a static file that normally lives at /static/index.html and would be served as via the assetbundle for /static/, but as that's an ugly uri to expose, I've set up a very simple resource to serve it via / too
  • /hello/: A very simple hello-world resource, check it out via swagger or just hit: /hello/text/there, /hello/text/world or /hello/text/you
  • /react/: Playing around with react
  • Having the different parts of the app live in separate subdirs like this allow new resources, like REST services to be hooked up everywhere else in the web scope, which is what I like to do, but it's also possible to do it the other way around and serve static files out of / and have the rest services limited to /api/, though that would conflict with swagger.
